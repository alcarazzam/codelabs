SQL (Structured Query Language) es un lenguaje de programación que permite consultar, modificar y añadir datos a una base de datos. Es utilizado por numerosos programas de bases de datos, como MySQL, PostgreSQL, SQLite, etc. Aunque es un estándar abierto, cada software de base de datos añade algunas funciones específicas que no están disponibles en el resto. En este curso nos basaremos en el estándar que admiten todas las bases de datos.

Debido al grado de abstracción que posee, se considera un lenguaje de alto nivel.

La última revisión del estándar fue realizada en 2016.

Tipos de lenguaje

Dentro de SQL, podemos diferenciar entre dos tipos de lenguajes principalmente:

  • Lenguaje de manipulación de datos (DML, por sus siglas en inglés Data Manipulation Language): permite consultar y manipular los datos de la base de datos.
  • Lenguaje de definición de datos (DDL, por sus siglas en inglés Data Definition Language): permite definir las bases de datos y sus tablas.

En la primera parte del curso trataremos la consulta de datos y, en la segunda, cómo crear y modificar esos datos.

La distinta información en las bases de datos está organizada en tablas. Cada base de datos puede tener un número ilimitado de tablas.

Cada tabla posee una o varias columnas de distintos tipos de datos que permiten organizar la información. Las tablas y columnas están identificadas por un nombre único. A cada una de las filas de una tabla se le conoce como registro.

Tipos de datos

Los tipos de datos permiten al programa de la base de datos saber qué pueden hacer con ellos. Estos son algunos de los tipos de datos más comunes que podemos encontrar:

varchar

Cadena de caracteres

int

Número entero

datetime

Fecha y hora

floar

Número decimal

Cada instrucción que se ejecuta se le conoce como sentencia y está formada por varios comandos, palabras clave. No importa si estas están escritas en mayúsculas o en minúsculas, pero para diferenciarlas del resto usaremos mayúsculas. Algunos editores de texto resaltan esta palabras con un color distinto para poder encontrarlas mejor. Las sentencias deben acabar con un punto y coma ; y pueden tener varias líneas.

Un ejemplo de sentencia SQL es el siguiente. Este muestra todas las filas de la tabla Customers.

SELECT * FROM Customers;
